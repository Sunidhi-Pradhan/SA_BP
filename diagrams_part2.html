7. Sequence Diagram – Login & Attendance Approval Flow

sequenceDiagram
    actor U as User
    participant LP as Login Page
    participant DB as Database
    participant GA as Google Authenticator
    participant EM as Email System
    participant AD as Admin Dashboard
    participant ASO as ASO Module
    participant APM as APM Module
    participant GM as GM Module
    participant HQSO as HQSO Module
    participant SD as SDHOD Module
    participant FN as Finance Module

    rect rgb(240, 253, 244)
    Note over U, GA: Authentication Flow
    U->>LP: Enter Employee ID & Password
    LP->>DB: Validate Credentials
    DB-->>LP: User Record
    alt Invalid Credentials
        LP-->>U: Error - Invalid ID or Password
    else Valid Credentials
        alt No 2FA Setup
            LP->>GA: Generate Secret & QR Code
            GA-->>U: Display QR Code to Scan
            U->>GA: Scan & Enter OTP
        else 2FA Already Setup
            LP->>U: Redirect to OTP Page
            U->>LP: Enter 6-digit OTP
        end
        LP->>GA: Verify OTP Code
        alt OTP Invalid
            GA-->>LP: Verification Failed
            LP->>DB: Increment Failed Attempts
            alt Attempts >= 5
                DB->>EM: Send Lockout Email
                LP-->>U: Account Locked for 2 Hours
            else Attempts < 5
                LP-->>U: Invalid Code - Try Again
            end
        else OTP Valid
            GA-->>LP: Verification Success
            LP->>DB: Reset Failed Attempts
            LP-->>U: Redirect to Role Dashboard
        end
    end
    end

    rect rgb(239, 246, 255)
    Note over AD, FN: Attendance Upload & Approval Workflow
    AD->>DB: Upload Attendance Excel
    DB-->>AD: Employees Imported
    ASO->>DB: Mark Daily Attendance
    ASO->>DB: Submit Monthly Report
    DB-->>ASO: Report Created

    ASO->>DB: ASO Approves Report
    DB-->>APM: Notify - Pending Approval

    APM->>DB: APM Approves Report
    DB-->>GM: Notify - Pending Approval

    GM->>DB: GM Approves Report
    DB-->>HQSO: Notify - Pending Approval

    HQSO->>DB: HQSO Approves Report
    DB-->>SD: Notify - Pending Approval

    SD->>DB: SDHOD Final Approval
    Note over SD: Status = COMPLETE

    SD->>DB: Generate LPC Document
    SD->>FN: Forward LPC to Finance
    FN->>DB: Approve LPC & Billing
    Note over FN: LPC Approved - Billing Done
    end
